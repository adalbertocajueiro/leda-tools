Prazos dos envios dos roteiros (todas as turmas):

  • Normal: 12 horas após a liberação.
  • Com atraso: 2 dias após a liberação.
  • Após os prazos acima o sistema NÃO recebe mais submissões.
  • Casos específicos com tempos diferenciados serão informados individualmente na própria descrição do roteiro.

Recomendações:

  • Baixar a versão mais recente do Eclipse (for Java EE Developers porque já vem com o plugin do Maven) e descompactar em sua conta do DSC. Usar essa versão em LEDA.
  • Não usar MacOS porque o plugin de compactação não funciona no Mac. Você precisa usar Windows ou Linux (mesmo que seja VM).
  • Caso queira usar o Maven na linha de comando:
    • Baixar o maven e descompactar em alguma pasta da sua conta
    • Abrir janela de comando (terminal) e ir para a pasta do projeto
    • Configurar a variavel de ambiente JAVA_HOME para a pasta onde se encontra o JDK instalado (deve ser o JDK 1.8). Voce pode ver a pasta do JDK nas preferencias do Eclipse "Java->Instaled JREs". No linux isso pode ser feito com o comando "export JAVA_HOME=pasta_instalacao_do_jdk"
    • Execute o comando no terminal: mvn install (para ignorar os testes basta usar mvn install -DskipTests)

Recomendações para as provas:

  • Verifique constantemente a hora atual do servidor, pois o recebimento de submissões após essa hora NÃO ACONTECERÁ!
  • As notas da prova serão publicadas e os alunos avisados. Até isso acontecer NÃO ENTREM EM CONTATO COM OS PROFESSORES para justificar nada, querer ver a prova ou querer saber porque deu erro de compilação. Será publicado no site o horário para revisão de provas e será comentado em sala sobre a solução!
  • Procure chegar uns 10 min mais cedo para verificar se sua conta no laboratório e a máquina está funcionando corretamente.
  • Todos os alunos deverão usar os computadores dos laboratórios. Quando for permitido o uso dos próprios laptops, os alunos serão avisados com antecedência.
  • Coindidências de código acarretarão em penalização severa independente de qualquer situação.
  • Pegar código pronto da internet é proibido! Durante a prova apenas serão admitidas duas janelas abertas na maquina do aluno: a do eclipse e a do PDF com a descrição da prova. Navegadores só serão permitidos para fazer o download da prova e para o envio das respostas. Programas como messengers, chats, skype e similares estão proibidos.
  • O download das provas de fora do DSC é permitido e o sistema registra todos os downloads realizados. Entretanto, o sistema não recebe submissões de prova vindas de computadores fora do DSC. No caso das provas, uma vez que o aluno faz o download, ele é automaticamente considerado fazendo a prova como se estivesse presente no laboratório. Portanto, se o aluno não vai enviar a prova, não faça o download!
  • Não será permitido pegar nenhum código pronto! Nos casos em que o aluno precise reutilizar algum código que implementou durante o semestre, o professor avisará com antecedência de 1 ou 2 dias.


Conferindo os envios: A qualquer momento as submissões podem ser conferidas das seguintes formas:

  • Aluno pode vizualizar o log dos envios na pasta do projeto (gerado automaticamente com a compilação maven).
  • O sistema também contém uma lista de submissões (roteiros e provas).
  • O sistema também contém uma lista de todas as submissões (incluindo roteiros de revisão).
